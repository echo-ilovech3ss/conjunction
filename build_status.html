Critical Build Failures Fixed

  • Removed Non-existent package

    Removed inter-font from packages list. This package does NOT exist in official Arch repos and would cause pacman to fail during ISO build.

    archiso/packages.x86_64

  • Removed Conflicting packages

    Removed power-profiles-daemon which directly conflicts with tlp. Both cannot coexist — pacman would refuse to install.

    archiso/packages.x86_64

  • Removed Duplicate packages

    Removed duplicate entries: mesa, lib32-mesa, rsync. Pacman handles this gracefully but it clutters the package list.

    archiso/packages.x86_64

  • Fixed Autologin user

    Changed --autologin conjunction to --autologin root. The live ISO has no "conjunction" user — login would fail with no shell, no desktop.

    archiso/airootfs/etc/systemd/system/getty@tty1.service.d/autologin.conf

Configuration & UI Improvements

  • Fixed Bootloader modes

    Changed bios.syslinux + uefi.systemd-boot (deprecated names) to the correct archiso bootmode strings: bios.syslinux.mbr, bios.syslinux.eltorito, uefi-x64.systemd-boot.esp, uefi-x64.grub.esp. Both BIOS and UEFI boot now work.

    archiso/profiledef.sh

  • Fixed SDDM configuration

    Removed invalid [GeneralDisplay] section, removed HideUsers=life (non-existent user), removed [X11] section (system uses Wayland). Config now only contains valid SDDM sections.

    archiso/airootfs/etc/sddm.conf.d/conjunction.conf

  • Fixed systemd service ordering

    Merged duplicate After= directives into one line. Systemd was dropping the first After=network-online.target because the second After=multi-user.target overwrote it.

    archiso/airootfs/etc/systemd/system/conjunction-setup.service

  • Fixed Desktop autostart entry

    Changed from sudo ... (fails in GUI) to konsole -e ... (opens terminal first). Removed unnecessary sudo since live environment autologins as root.

    archiso/airootfs/etc/skel/.config/autostart/conjunction-install.desktop

Installer & Build Script Fixes

  • Fixed Installer file copy logic

    The installer tried to find engine.py, cli.py, and setup_conjunction_ui.sh in its own directory (/usr/local/bin/). These files are actually at /opt/conjunction/ on the ISO. Now copies from the correct path.

    archiso/airootfs/usr/local/bin/conjunction-installer.sh

  • Fixed Premature file copy

    Installer tried to copy conjunction-post-install.sh before it was created (it's generated by a heredoc later in the same script). Removed the premature copy that always failed silently.

    archiso/airootfs/usr/local/bin/conjunction-installer.sh

  • Added Application files to ISO

    Build script now copies engine.py, cli.py, and setup_conjunction_ui.sh into the ISO's /opt/conjunction/ directory, so the installer can find and deploy them.

    build-iso.sh
