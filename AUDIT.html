✅ What Was Fixed

Fixed Live ISO had no desktop session archiso/

SDDM was configured without autologin, and no live user existed. The ISO booted to a blank screen or login prompt in Hyper-V.

  • Created customize_airootfs.sh that creates a conjunction live user with passwordless sudo
  • Added [Autologin] section to sddm.conf targeting the conjunction user
  • Enabled SDDM, NetworkManager, and the live setup service during image build

Fixed Wayland default broke Hyper-V/VM display conjunction.conf

SDDM defaulted to Wayland which often has no working display driver in Hyper-V Generation 2 VMs.

  • Changed DisplayServer=wayland to DisplayServer=x11
  • Added plasma-x11-session package to ISO so the X11 session .desktop file exists

Fixed Installer broken on NVMe/virtio disks conjunction-installer.sh

The installer hardcoded partition names like /dev/sda1. NVMe uses /dev/nvme0n1p1 and virtio uses /dev/vda1.

  • Added partition_path() helper that detects disk naming and generates correct partition paths
  • Added sync; partprobe; udevadm settle after partitioning so kernel sees new partitions
  • Added ${TARGET_DISK#/dev/} to strip leading /dev/ if user pastes the full path

Fixed Missing critical packages packages.x86_64

The installer script needed packages not in the ISO image.

  • Added arch-install-scripts (provides pacstrap, genfstab, arch-chroot)
  • Added parted (partitioning), gptfdisk (GPT tools)
  • Added inetutils (provides ping, hostname)
  • Added inter-font (valid Arch Extra package, replaces non-existent ttf-inter)
  • Added openssh, bluez-utils, cups (needed by services the installer enables)

Fixed Invalid/removed packages packages.x86_64

Several packages were renamed, removed from Arch repos, or conflicted with each other.

  • Removed ttf-inter, otf-inter (replaced with inter-font)
  • Removed kvantum-theme-breeze (does not exist in official repos)
  • Removed latte-dock (removed from Arch repos; using plank instead)

Fixed Build script preserved stale output build-iso.sh

The out/conjunction-profile/ directory carried deleted or obsolete files between builds, causing ISOs to contain old bugs.

  • Added rm -rf "${PROFILE_DIR}" before each build so the generated profile always matches source
  • Fixed ISO verification to search by glob when the expected filename doesn't match

Fixed KDE Plasma 5 vs 6 tool mismatch setup_conjunction_ui.sh

The UI setup script hardcoded kwriteconfig5 and qdbus. Arch now ships Plasma 6 where these are named kwriteconfig6 / qdbus6.

  • Added KWRITECONFIG and KDBUS_CMD detection that checks for both 5 and 6 versions
  • Replaced all kwriteconfig5 calls with "$KWRITECONFIG"
  • Replaced all qdbus calls with "$QDBUS_CMD"

Fixed Flatpak install used wrong app ID cli.py

flatpak search output varies by locale. Taking the first column grabbed the display name, not the application ID (e.g., org.mozilla.firefox).

  • Now iterates columns and prefers the value containing a dot (the application ID format)

Fixed Setup service caused boot delays conjunction-setup.service

The live setup service depended on network-online.target, which could take 30+ seconds or never resolve in VMs without DHCP.

  • Changed to depend on NetworkManager.service (just needs the service running, not connectivity)
  • Network connectivity is probed asynchronously inside the script itself

Fixed Post-install service never disabled conjunction-installer.sh

conjunction-post-install.service ran on every boot with RemainAfterExit=yes, replaying the boot animation and sysctl writes every time.

  • Added systemctl disable conjunction-post-install.service at the end of the script

Fixed UEFI/BIOS auto-detection conjunction-installer.sh

The installer asked for UEFI vs BIOS partitioning. Users often picked wrong.

  • Auto-detects firmware via /sys/firmware/efi and defaults to GPT or MBR accordingly

Fixed Desktop shortcuts inconsistent archiso/airootfs

The .config/autostart entry lacked sudo (installer needs root), and no ~/Desktop shortcut existed.

  • Added sudo to the autostart desktop entry
  • Created etc/skel/Desktop/conjunction-install.desktop for visible desktop shortcut
  • Both entries now use konsole -e sudo ... which is correct

Fixed pacman install flag syntax cli.py

The force reinstall code assembled -S --overwrite '*' as a single string instead of separate arguments.

  • Now uses proper list: ["pacman", "-S", "--noconfirm", "--overwrite", "*"]

Fixed Build artifacts in git .gitignore

Generated ISO profiles, Python cache, ISO files, and logs were untracked but visible in the repo.

  • Added .gitignore for __pycache__/, out/, *.iso, *.log

🐛 Remaining Bugs

Critical RuntimeError shadowed in engine.py engine.py:125

class RuntimeError(ConjunctionError):

This shadows Python's built-in RuntimeError. Any code in this module (or modules importing from it) that catches or raises RuntimeError will hit the custom class instead. The custom class should be renamed to something like RuntimeInstallError.

Critical PulseAudio socket path hardcoded to UID 1000 engine.py:868-872

env["PULSE_SERVER"] = "unix:/run/user/1000/pulse/native"

If the user's UID is not 1000 (e.g., conjunction created after root has UID 1000, or multi-user systems), audio will fail silently. Should use /run/user/$(id -u)/pulse/native or the PULSE_SERVER environment variable if already set.

Critical Setup service runs on every boot conjunction-setup.service

The service is enabled by customize_airootfs.sh and has no mechanism to disable itself after first run. On a live ISO this is fine (ephemeral), but if the service file is accidentally included in an installed system, it will run the live setup script on every boot.

Critical pacman -Sy partial upgrade in setup UI setup_conjunction_ui.sh:128

run_sudo pacman -Sy --noconfirm

-Sy syncs the database but doesn't upgrade. Running -S after -Sy on an Arch system is a partial upgrade and can break the system due to library mismatches. Should use -Syu or just -S (database is already synced from ISO build).

High Logging handler accumulation engine.py:76-99

_setup_logging() is called at module import time and adds handlers to the "conjunction-runner" logger. If the module is reloaded (e.g., in testing or interactive sessions), handlers accumulate and each message is printed/logged multiple times. Should check if not logger.handlers before adding.

High chmod -R 755 airootfs/ breaks sensitive files build-iso.sh:228

Recursively sets 755 on all files under airootfs/, including /etc/shadow which should be 0400. While profiledef.sh file_permissions overrides this, the chmod runs before mkarchiso processes the profile, and the permission override only works if the file exists at the expected path.

High Installer enables SSH with no auth config conjunction-installer.sh:299

systemctl enable sshd opens port 22 with password authentication. On a fresh install with no firewall, this is a security exposure. Should disable password auth (PasswordAuthentication no) or not enable sshd by default.

High Boot animation plays during systemd boot conjunction-installer.sh:371-417

The post-install service runs a 5+ second shell animation before any actual configuration. This delays boot and could race with display manager startup. The animation should be decoupled from the service or run only on first login.

High chpasswd uses colon-delimited format conjunction-installer.sh:268

echo "${USERNAME}:${PASSWORD}" | arch-chroot /mnt chpasswd

chpasswd by default expects user:password colon-delimited input. However, if the password contains a colon character, it will be truncated. Should use echo "$USERNAME:$PASSWORD" | arch-chroot /mnt chpasswd with --crypt-method or use arch-chroot /mnt usermod --password for proper hash-based password setting.

High No Flatpak remote configured archiso/packages.x86_64, cli.py

flatpak is installed but no remote (e.g., flathub) is added. The cj install command will fail on every Flatpak search because there are no configured remotes to search.

High yay not in ISO packages packages.x86_64

The CLI's AUR fallback depends on yay, but it's not installed. Users must install it manually. The README doesn't mention this gap.

High img.size = largest is wrong engine.py:348

img.size = largest

Pillow's Image.size is a read-only property. This line does nothing. The icon is saved at its original size, not the largest available. Should use img = img.resize(largest) or img.thumbnail(largest).

High Installer disk detection shows sr/ram devices conjunction-installer.sh:53

lsblk -d -o NAME,SIZE,MODEL,ROTA | grep -v "loop\|sr\|ram"

Missing zram and dm- (device-mapper) from the exclusion list. Users could accidentally select a zram device or LVM logical volume as the install target.

Medium lib32-* packages may not exist packages.x86_64:105-109

Packages like lib32-glibc, lib32-gcc-libs, lib32-libpng, lib32-libxml2, lib32-ncurses are in the multilib repo, but the ISO's pacman.conf needs multilib enabled. The build script enables it via sed, but if the sed fails silently, the ISO build will fail or these packages will be missing.

Medium wl-clipboard in X11 ISO packages.x86_64:198

wl-clipboard is Wayland-only. The live session defaults to X11. Clipboard functions using wl-copy/wl-paste won't work. Should include xclip or xsel instead/in addition.

Medium plymouth not configured packages.x86_64:199

plymouth is installed but no Plymouth theme is set up and no mkinitcpio hook is configured. It does nothing useful and may cause boot delays waiting for a daemon that isn't started.

Medium Single-disk systems still prompt for disk conjunction-installer.sh:56

When only one disk exists, the installer should auto-select it instead of asking. This is a UX issue that confuses non-technical users.

Medium No LVM/RAID/swap detection conjunction-installer.sh

The installer unmounts /dev/${TARGET_DISK}* but doesn't check for LVM logical volumes, RAID arrays, or active swap partitions on those partitions. These may prevent unmounting and partitioning.

Medium Snapper config race with GRUB conjunction-installer.sh:305-328

Snapper is configured after GRUB is installed. grub-btrfs won't detect any snapshots at GRUB install time, so the GRUB menu won't show snapshot entries until the next grub-mkconfig run.

Medium Font download URL hardcoded setup_conjunction_ui.sh:381

The Inter font download URL (https://github.com/rsms/inter/releases/download/v4.0/Inter-4.0.zip) is hardcoded. When Inter releases v4.1+, this URL may break. Should pin to a specific release or use the package manager (already installed via inter-font).

Medium DXVK_ASYNC may not exist engine.py:821

env["DXVK_ASYNC"] = "1"

Standard DXVK does not support the DXVK_ASYNC env var. Only the patched dxvk-async fork does. Setting it on standard DXVK has no effect but may confuse users into thinking async compilation is active.

Medium Wine update step is a no-op cli.py:451-477

The "Step 4/5: Checking Wine components" in cj update runs wine --version and winetricks --version but never actually updates anything. It just prints the version and says "checked." Should either update or be removed.

Medium Snapshot ID parsing is fragile cli.py:399-407

Snapper output format varies by version and locale. The code takes the last line of snapper list --columns number, but the header line and potential empty lines make this unreliable. Should parse structured output or use --columns more carefully.

⚠️ Warnings / Technical Debt

Low Shell aliases conflict with macOS-like functions setup_conjunction_ui.sh:588-589

  • alias reboot="echo 'Use sudo reboot'" conflicts with the open() function that wraps xdg-open
  • alias vim="nvim 2>/dev/null || vim" — nvim is not in the ISO package list

Low No test suite exists

  • No unit tests for engine.py or cli.py
  • No integration tests for the installer
  • No smoke tests for the ISO boot

Low Docker build not tested locally

  • The build script runs docker run --privileged which requires Docker Desktop with Linux containers
  • No CI/CD pipeline exists for automated builds
  • The existing ISO in out/ was built before most fixes

Low No Secure Boot consideration

  • The ISO doesn't ship with signed bootloaders
  • Secure Boot must be disabled in UEFI settings
  • Should be documented or handled with shim/systemd-boot signed binaries

Low No rollback mechanism on failed install

  • If the installer fails midway (e.g., pacstrap fails due to mirror issues), the disk is wiped with no recovery
  • No backup prompt before destructive operations

Low UI setup not auto-run after install

  • README says: "Run UI setup: sudo /opt/conjunction/setup_conjunction_ui.sh"
  • Should be triggered automatically during first login or via a welcome app

Low conjunction-post-install.service uses RemainAfterExit=yes

  • Combined with Type=oneshot, this means systemd considers the service "active" even after it exits
  • Should use Type=oneshot without RemainAfterExit for a true one-shot service

Low No internationalization (i18n)

  • All UI strings are English-only
  • Installer prompts have no translation support
  • Hardcoded for now but blocks non-English users

Low ISO size bloat

  • go, nodejs, npm, gcc, cmake, ninja, clang, llvm are all development tools — adds ~1GB+ to ISO
  • These should be optional post-install, not on the live media
  • nmap is a network scanning tool — not needed for a macOS-like desktop

Low nvidia-dkms in ISO but nouveau not blacklisted

  • NVIDIA proprietary driver is installed, but no udev rule or modprobe config blacklists nouveau
  • Both drivers may load simultaneously, causing display issues

🔍 How to Audit This Project

Guide Static Analysis (do this first)

  • Shell scripts: Run bash -n script.sh to check syntax. Use shellcheck for deeper analysis (not installed by default on Windows).
  • Python: Run python -m py_compile file.py for syntax. Use mypy for type checking.
  • Package validity: Search https://archlinux.org/packages/?q=PACKAGE_NAME for every package in packages.x86_64. If it returns no results, the ISO build will fail.
  • Service dependencies: For every systemctl enable X, verify package X is in the installed package list. Cross-reference with pacstrap call in installer.

Guide Runtime Testing (requires Linux)

  • Build the ISO: Run ./build-iso.sh in a Linux environment with Docker. Watch for pacman errors about missing packages.
  • Test in QEMU: qemu-system-x86_64 -m 4G -enable-kvm -cdrom out/conjunction-*.iso. Verify live desktop appears, installer launches, and disk can be partitioned.
  • Test in Hyper-V: Create a Gen 2 VM, disable Secure Boot, attach ISO, boot. Verify X11 session, autologin, and installer.
  • Test installer flow: Let the installer partition a virtual disk. Check that GRUB installs, services are enabled, and reboot works.
  • Test post-install: After reboot, verify cj command exists, setup_conjunction_ui.sh runs without errors, and the desktop looks correct.

Guide Common Bug Patterns to Search For

  • grep -r "systemctl enable" archiso/ — cross-reference every enabled service with packages.x86_64
  • grep -r "1000\|/run/user" engine.py cli.py — find hardcoded UIDs
  • grep -rn "\${TARGET_DISK}" archiso/ — verify every partition path uses the partition_path() helper
  • grep -rn "kwriteconfig\|qdbus " setup_conjunction_ui.sh — verify all KDE tool calls use the detected variable
  • grep -rn "class.*Error" engine.py — check for shadowed builtins
  • grep -rn "\.size\s*=" engine.py — look for incorrect Pillow attribute usage
  • python -m py_compile *.py — catch syntax errors before they reach the ISO
  • bash -n *.sh archiso/**/*.sh — catch shell syntax errors
